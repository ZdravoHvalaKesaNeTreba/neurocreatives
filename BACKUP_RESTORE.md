# Восстановление из бэкапа Neurocreatives v1.0

Этот документ описывает процедуру восстановления проекта Neurocreatives из архивного бэкапа версии 1.0.

## 📦 Информация о бэкапе

- **Версия**: 1.0
- **Дата создания**: 17 марта 2026
- **Файл бэкапа**: `neurocreatives-v1.0-20260317-135625.tar.gz`
- **Размер**: ~158 KB
- **Местоположение**: `backups/`

## 🔍 Что включено в бэкап

Бэкап содержит весь исходный код проекта, за исключением:
- `venv/` - виртуальное окружение Python
- `__pycache__/` - кэш Python
- `*.pyc` - скомпилированные файлы Python
- `.DS_Store` - системные файлы macOS
- `downloads/` - скачанные изображения из Telegram
- `backups/` - другие бэкапы

## 📋 Предварительные требования

Перед восстановлением убедитесь, что у вас установлены:
- Python 3.10+ 
- PostgreSQL 14+ (опционально, если используете локальную БД)
- Git (опционально)

## 🔧 Процедура восстановления

### Шаг 1: Подготовка директории

Создайте новую директорию для восстановления проекта:

```bash
mkdir neurocreatives-restored
cd neurocreatives-restored
```

### Шаг 2: Извлечение файлов из бэкапа

Скопируйте файл бэкапа в текущую директорию и извлеките содержимое:

```bash
# Если бэкап находится в родительской директории
cp ../neurocreatives/backups/neurocreatives-v1.0-*.tar.gz .

# Извлеките архив
tar -xzf neurocreatives-v1.0-*.tar.gz

# Проверьте содержимое
ls -la
```

### Шаг 3: Создание виртуального окружения

Создайте и активируйте виртуальное окружение Python:

```bash
# Создание виртуального окружения
python3 -m venv venv

# Активация (macOS/Linux)
source venv/bin/activate

# Активация (Windows)
# venv\Scripts\activate
```

### Шаг 4: Установка зависимостей

Установите все необходимые пакеты из requirements.txt:

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### Шаг 5: Настройка переменных окружения

Скопируйте файл примера конфигурации и заполните его:

```bash
cp .env.example .env
```

Отредактируйте `.env` файл, заполнив необходимые параметры:

```bash
# Используйте ваш любимый редактор
nano .env
# или
code .env
```

**Обязательные параметры:**

- `DATABASE_URL` - URL подключения к PostgreSQL
- `TELEGRAM_API_ID` - ID API Telegram
- `TELEGRAM_API_HASH` - Hash API Telegram  
- `OPENAI_API_KEY` - ключ API OpenAI
- `CHANNELS_TO_PARSE` - список каналов для парсинга

### Шаг 6: Создание директории для загрузок

```bash
mkdir downloads
```

### Шаг 7: Инициализация базы данных (если используете новую БД)

Если вы создаёте новую базу данных:

```bash
# Создайте базу данных PostgreSQL
createdb neurocreatives

# Или через psql:
# psql -U postgres
# CREATE DATABASE neurocreatives;
```

База данных будет автоматически инициализирована при первом запуске приложения.

### Шаг 8: Запуск приложения

Запустите локальный сервер:

```bash
python run.py
```

Или используйте прямой запуск:

```bash
./venv/bin/python run.py
```

Приложение будет доступно по адресу: **http://localhost:8000**

## ✅ Проверка работоспособности

После запуска проверьте следующее:

1. **Веб-интерфейс**: Откройте http://localhost:8000 в браузере
2. **Вкладки**: Проверьте работу вкладок "Галерея" и "Таблица"
3. **Фильтры**: Проверьте применение фильтров в таблице
4. **Детальная панель**: Кликните на креатив для просмотра деталей
5. **Закрытие модального окна**: Кликните вне панели для закрытия

## 🔄 Восстановление данных (опционально)

Если у вас есть дамп базы данных, восстановите его:

```bash
# Из SQL дампа
psql -U postgres neurocreatives < backup.sql

# Или используя pg_restore для кастомного формата
pg_restore -U postgres -d neurocreatives backup.dump
```

## 🐛 Решение проблем

### Ошибка подключения к PostgreSQL

```bash
# Проверьте доступность базы данных
psql -U your_username -d neurocreatives

# Проверьте DATABASE_URL в .env файле
```

### Ошибки с SSL сертификатом

Если используете Yandex Cloud PostgreSQL, скачайте сертификат:

```bash
mkdir -p ~/.postgresql
cd ~/.postgresql
curl -O https://storage.yandexcloud.net/cloud-certs/CA.pem
mv CA.pem root.crt
```

### Не устанавливаются зависимости

```bash
# Обновите pip
pip install --upgrade pip

# Переустановите зависимости
pip install -r requirements.txt --force-reinstall
```

## 📝 Примечания

- Бэкап **не включает** скачанные изображения из директории `downloads/`
- Бэкап **не включает** конфиденциальный `.env` файл с учетными данными
- База данных **не входит** в бэкап - создайте её отдельный дамп при необходимости
- Виртуальное окружение **не входит** в бэкап - создаётся заново
